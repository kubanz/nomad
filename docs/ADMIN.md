# Админ-панель

Панель управления сайтом: `https://nomad-transfer.com/admin`

## Возможности (текущая версия)

- Вход по логину и паролю, выход, сессии на 12 часов.
- Пользователи: создание, редактирование, блокировка, удаление, смена пароля.
- Группы и матрица доступов: «ресурс × действие», назначение групп пользователям.
- Заготовленные разделы «Страницы» и «Новости» — права на них уже настраиваются,
  сами разделы будут реализованы следующими этапами.

## Первый запуск

1. Задайте секрет в окружении (локально — `.env.local`, на Vercel — Settings → Environment Variables):

   ```
   ADMIN_AUTH_SECRET=<случайная строка 32+ символов>
   ```

   Сгенерировать: `node -e "console.log(crypto.randomBytes(32).toString('hex'))"`

   ⚠️ При смене секрета все сессии сбросятся, а данные админки в Vercel Blob
   станут нечитаемыми (они зашифрованы этим секретом).

2. Откройте `/admin` — пока нет ни одного пользователя, откроется страница
   первичной настройки `/admin/setup`, где создаётся первый администратор.
   После появления первого пользователя страница настройки отключается.

## Матрица доступов

Право — это пара «ресурс : действие». Группа содержит набор прав,
пользователь получает объединение прав всех своих групп.

| Ресурс         | Действия                                              |
| -------------- | ----------------------------------------------------- |
| Пользователи   | просмотр, создание, редактирование, удаление          |
| Группы и права | просмотр, создание, редактирование, удаление          |
| Страницы       | просмотр, создание, редактирование, удаление, публикация |
| Новости        | просмотр, создание, редактирование, удаление, публикация |
| Медиафайлы     | просмотр, создание, удаление                          |
| Настройки      | просмотр, редактирование                              |

Особые правила:

- `*` — полный доступ (у системной группы «Администраторы», её нельзя удалить
  или изменить её права).
- Нельзя заблокировать/удалить собственную учётную запись и менять собственные группы.
- Смена пароля или блокировка сбрасывает активные сессии пользователя.

Группы по умолчанию: «Администраторы» (полный доступ) и «Редакторы»
(страницы/новости без публикации и удаления). Новые группы создаются в
разделе «Группы и права» с любым набором галочек в матрице.

## Где хранятся данные

Отдельной БД нет. Данные (пользователи, группы) — JSON-документы:

- **Локально / Docker**: файлы в `.data/admin/` (в git не попадают).
- **Vercel (продакшен)**: Vercel Blob (`admin-data/*.json`), содержимое
  зашифровано AES-256-GCM ключом из `ADMIN_AUTH_SECRET`, т.к. блобы публичны по URL.

Драйвер выбирается автоматически: в dev — файлы, в продакшене — Blob (если задан
`BLOB_READ_WRITE_TOKEN`). Принудительно: `ADMIN_STORE=fs` или `ADMIN_STORE=blob`.

## Как устроена защита

- `middleware.ts` — не пускает на `/admin/*` без валидной куки сессии
  (подписанный HMAC-SHA256 токен, httpOnly, secure).
- `lib/admin/auth.ts` → `getAuth()` — полная серверная проверка: подпись, срок,
  блокировка, версия сессии.
- Каждый серверный экшен вызывает `requirePermission(ресурс, действие)` —
  права проверяются на сервере, а не только в интерфейсе.
- Пароли — scrypt (N=16384), `/admin` закрыт в `robots.txt` и `noindex`.

## Как добавить новый раздел в админку

1. Добавьте ресурс в `RESOURCES` и его действия в `RESOURCE_ACTIONS`
   (`lib/admin/types.ts`, `lib/admin/rbac.ts`) + подписи в `RESOURCE_LABELS`.
2. Создайте страницы в `app/admin/(panel)/<раздел>/` — в начале страницы
   проверка `can(auth.permissions, "<ресурс>", "view")`.
3. В серверных экшенах раздела вызывайте `requirePermission("<ресурс>", "<действие>")`.
4. Добавьте пункт в `NAV_ITEMS` (`app/admin/(panel)/layout.tsx`) — он автоматически
   скроется у пользователей без права просмотра.
